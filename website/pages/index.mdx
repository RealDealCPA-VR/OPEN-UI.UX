# OpenCodex

**Mission Control for AI coding agents — a standalone desktop app that lives next to your editor and drives Claude Code, Aider, OpenCode, and a built-in agent over any LLM.**

![Subagent tree view — OpenCodex Mission Control](/hero-subagent-tree.svg)

> The image above is a hand-drawn SVG placeholder. The shipped landing hero is a real screenshot of the AgentRunDrawer subagent tree view (`apps/desktop/src/renderer/components/AgentRunDrawer.tsx`) — see `PLACEHOLDERS.md` in the repo for the replacement spec.

OpenCodex is an open-source, local-first, MIT-licensed desktop app that sits _above_ the coding-agent CLIs you already use, unifies their approval and merge-review UX, and is provider-agnostic from day one. It does not replace your editor — it drives the agents that work in it.

## The three pillars

- **Runners** — drive the built-in agent, Claude Code, Aider, OpenCode, or any runner shipped via a plugin. The subagent tree view (mocked in the hero above; lives in `apps/desktop/src/renderer/components/AgentRunDrawer.tsx`) shows every parallel worker live: status, tokens, cost, and the diff each one is staging in its own worktree.
- **Multi-agent orchestration** — fan out subtasks to parallel workers, each with its own model, scope, and git worktree, then accept or reject merges per worker.
- **MCP + plugins** — every MCP server you connect becomes a tool / resource / prompt source. Third-party packages contribute tools, providers, runners, and full UI panels via the plugin SDK.

## Provider-agnostic as a feature, not a footnote

Eight providers in the box — OpenAI, Anthropic, Google Gemini, xAI Grok, Mistral, Ollama (local), OpenRouter, Voyage (embed-only) — all behind a single `LLMProvider` contract. Switch mid-conversation. The agent loop, the tool layer, the UI never name a specific vendor. Adding a new provider is one package and one interface implementation; ship it as a plugin or upstream it.

## What it is, at a glance

- **Electron + TypeScript** monorepo. Native app on macOS, Windows, and Linux.
- **Any LLM**: OpenAI, Anthropic, Google, xAI, Mistral, Ollama, OpenRouter — all behind a single `LLMProvider` interface.
- **Agentic**: a tool registry with permission tiers (`read` / `write` / `execute` / `network`) and an approval system for every mutation.
- **RAG over your codebase**: Tree-sitter chunker + hybrid retrieval (LanceDB + SQLite FTS5).
- **MCP-native**: Model Context Protocol servers are first-class tools.
- **Plugin SDK**: tools, providers, runners, and UI panels can be contributed by third-party packages.
- **No backend**: API keys live in the OS keychain via `keytar`. Everything runs locally.
