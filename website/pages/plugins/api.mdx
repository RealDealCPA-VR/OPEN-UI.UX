## `PluginManifest`

The Zod-validated shape of every `opencodex.plugin.json`:

```ts
const ManifestSchema = z.object({
  name: z.string().min(1),
  version: z.string().min(1),
  displayName: z.string().min(1),
  description: z.string().optional(),
  author: z.string().optional(),
  license: z.string().optional(),
  homepage: z.string().url().optional(),
  entry: z.string(), // relative path to the JS entry
  engines: z.object({ opencodex: z.string() }), // semver range
  permissions: z.array(PermissionSchema).default([]),
  contributions: ContributionSchema.default({}),
});

type PluginManifest = z.infer<typeof ManifestSchema>;
```

### Field reference

| Field               | Type             | Notes                                                                                |
| ------------------- | ---------------- | ------------------------------------------------------------------------------------ |
| `name`              | `string` (min 1) | Unique plugin id within the install. Lowercase + hyphens.                            |
| `version`           | `string` (min 1) | Semver.                                                                              |
| `displayName`       | `string` (min 1) | Shown in the Plugins panel.                                                          |
| `description`       | `string?`        | One-line summary.                                                                    |
| `author`            | `string?`        | Free-form.                                                                           |
| `license`           | `string?`        | SPDX id (e.g. `MIT`).                                                                |
| `homepage`          | `url?`           | Validated as a URL if present.                                                       |
| `entry`             | `string`         | Path relative to plugin root pointing at the module that default-exports a `Plugin`. |
| `engines.opencodex` | `string`         | Semver range of OpenCodex versions this plugin supports.                             |
| `permissions`       | `Permission[]`   | Defaults to `[]`.                                                                    |
| `contributions`     | `Contribution`   | Defaults to `{}`.                                                                    |

## `Permission`

```ts
type Permission =
  | 'workspace.read'
  | 'workspace.write'
  | 'shell.execute'
  | 'network.fetch'
  | 'settings.read'
  | 'settings.write'
  | 'ui.panel';
```

| Permission        | What it unlocks                                |
| ----------------- | ---------------------------------------------- |
| `workspace.read`  | Register `permissionTier: 'read'` tools.       |
| `workspace.write` | Register `permissionTier: 'write'` tools.      |
| `shell.execute`   | Register `permissionTier: 'execute'` tools.    |
| `network.fetch`   | Register `permissionTier: 'network'` tools.    |
| `settings.read`   | `host.getSetting(key)` returns values.         |
| `settings.write`  | `host.setSetting(key, value)` persists.        |
| `ui.panel`        | Manifest may declare `contributions.panels[]`. |

## `Contribution`

```ts
const ContributionSchema = z.object({
  tools: z.array(z.string()).optional(),
  providers: z.array(z.string()).optional(),
  panels: z
    .array(
      z.object({
        id: z.string(),
        title: z.string(),
        entry: z.string(),
      }),
    )
    .optional(),
  slashCommands: z
    .array(
      z.object({
        name: z.string(),
        entry: z.string(),
      }),
    )
    .optional(),
});
```

| Field           | Element shape          | Notes                                                                                                  |
| --------------- | ---------------------- | ------------------------------------------------------------------------------------------------------ |
| `tools`         | `string`               | Names of tools this plugin registers via `host.registerTool()`. Must match the registered `Tool.name`. |
| `providers`     | `string`               | Names of `ProviderFactory.id`s this plugin registers.                                                  |
| `panels`        | `{ id, title, entry }` | `entry` is the HTML file (relative to plugin root). Loaded into a sandboxed iframe.                    |
| `slashCommands` | `{ name, entry }`      | `entry` resolves a handler — surfaced as `/<name>` in the chat composer.                               |

## `PluginHost`

```ts
interface PluginHost {
  readonly pluginId: string;
  readonly logger: PluginLogger;

  registerTool(tool: Tool): void;
  registerProvider(provider: ProviderFactory): void;
  registerSlashCommand(name: string, handler: (args: string) => Promise<void>): void;

  getSetting<T = unknown>(key: string): Promise<T | undefined>;
  setSetting<T = unknown>(key: string, value: T): Promise<void>;
}
```

| Member                                | Description                                                                                                        |
| ------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| `pluginId`                            | Echoes `manifest.name`. Useful for namespacing keys.                                                               |
| `logger`                              | Pino-shaped: `info(msg, meta?)`, `warn(...)`, `error(...)`. Output goes to the host's log file.                    |
| `registerTool(tool)`                  | Adds a `Tool` to the shared `ToolRegistry` as `plugin__<id>__<name>`. Unregistered on `deactivate()` or uninstall. |
| `registerProvider(provider)`          | Adds a `ProviderFactory` to the host registry.                                                                     |
| `registerSlashCommand(name, handler)` | Surface a `/`-command in the chat composer.                                                                        |
| `getSetting(key)`                     | Reads from the plugin's namespaced settings bucket. Requires `settings.read`. Returns `undefined` if unset.        |
| `setSetting(key, value)`              | Persists into the plugin's namespaced settings bucket. Requires `settings.write`.                                  |

## `Plugin`

```ts
interface Plugin {
  activate(host: PluginHost): void | Promise<void>;
  deactivate?(): void | Promise<void>;
}
```

`activate(host)` runs once when the plugin is enabled. Register tools/providers/etc. here.
`deactivate()` is best-effort cleanup on disable/uninstall. Host already auto-unregisters anything you registered through the `PluginHost` methods, so this hook is for your own resources (timers, child processes, etc.).

---

## `definePlugin(plugin)`

```ts
function definePlugin(plugin: Plugin): Plugin;
```

Identity function with the type narrowed. Always use this for the default export so the TypeScript shape is checked at build time:

```ts
import { definePlugin } from '@opencodex/plugin-sdk';

export default definePlugin({
  async activate(host) {
    /* ... */
  },
});
```

## Loader helpers (host-side)

These are exported for the host to load plugins from disk. Plugin authors rarely call them directly.

### `readManifest(pluginPath)`

```ts
function readManifest(pluginPath: string): Promise<PluginManifest>;
```

Reads `<pluginPath>/opencodex.plugin.json`, JSON.parses, and runs `ManifestSchema.safeParse()`. Throws `PluginLoadError` on any failure.

### `loadPluginModule(pluginPath, manifest)`

```ts
function loadPluginModule(pluginPath: string, manifest: PluginManifest): Promise<Plugin>;
```

Dynamically imports the entry module, locates the default `Plugin` export, validates it has an `activate` function. Throws `PluginLoadError` on failure.

### `PluginLoadError`

```ts
class PluginLoadError extends Error {
  readonly pluginPath: string;
}
```

Thrown by both loader helpers when manifest validation, JSON parsing, or module import fails. `pluginPath` is the disk path so the host can surface it in the install UI.

## Runner SDK

The runner SDK lets a plugin contribute a new entry to the agent-runner
picker. See the [Agent runners guide](../guides/runners/) for the
user-facing perspective and [Building a runner adapter](./authoring/#building-a-runner-adapter)
for end-to-end author docs.

### Imports

```ts
import {
  type SubagentRunner,
  type SubagentRunOptions,
  type ChatEvent,
  type RunnerInstallStatus,
  RunnerRegistry,
  RunnerAlreadyRegisteredError,
} from '@opencodex/core';
```

All runner types live in `@opencodex/core` — not `@opencodex/plugin-sdk` —
because the host's agent loop also depends on them. The plugin SDK
re-exports the `host.registerRunner(...)` method below.

---

### `host.registerRunner(runner)`

```ts
registerRunner(runner: SubagentRunner): void
```

Adds a runner to the shared `RunnerRegistry`. Requires the `agent.runner`
permission in the manifest. Throws `RunnerAlreadyRegisteredError` if a
runner with the same `id` is already registered (built-in or by another
plugin).

Auto-unregistered on `deactivate()` or uninstall.

```ts
host.registerRunner({
  id: 'my-runner',
  displayName: 'My Runner',
  streaming: true,
  async checkInstalled() {
    /* ... */
  },
  async *run(options) {
    /* yield ChatEvents */
  },
});
```

---

### `SubagentRunner`

```ts
interface SubagentRunner {
  readonly id: string;
  readonly displayName: string;
  readonly streaming: boolean;
  checkInstalled(): Promise<RunnerInstallStatus>;
  run(options: SubagentRunOptions): AsyncIterable<ChatEvent>;
}
```

| Field              | Type                           | Notes                                                                                                                                         |
| ------------------ | ------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `id`               | `string`                       | Unique across all runners. Used as the dropdown value and persisted in scheduled tasks. Convention: lowercase + hyphens (e.g. `claude-code`). |
| `displayName`      | `string`                       | Shown in the runner picker and Settings → Runners list.                                                                                       |
| `streaming`        | `boolean`                      | `false` makes the UI show a spinner instead of a live text bubble. Set `false` for runners that only report on exit (Aider).                  |
| `checkInstalled()` | `Promise<RunnerInstallStatus>` | Probed by Settings and at the start of every run. Keep under ~3 seconds.                                                                      |
| `run(options)`     | `AsyncIterable<ChatEvent>`     | Must yield exactly one terminal `done` event. Honor `options.signal`.                                                                         |

---

### `SubagentRunOptions`

```ts
interface SubagentRunOptions {
  runId: string;
  prompt: string;
  cwd: string;
  model?: string;
  allowedTools?: readonly string[];
  signal: AbortSignal;
  logger: PluginLogger;
  cliPath?: string;
}
```

| Field          | Type                 | Notes                                                                                                                        |
| -------------- | -------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| `runId`        | `string`             | Stable id for this run. Use as the prefix in your log filename.                                                              |
| `prompt`       | `string`             | The user's prompt (or, for scheduled tasks, the cron task's prompt body).                                                    |
| `cwd`          | `string`             | Always a git worktree path for external runners; the workspace root for the `internal` runner.                               |
| `model`        | `string?`            | Provider model id when the runner cares (Claude Code, OpenCode). `undefined` for runners that use their own selector.        |
| `allowedTools` | `readonly string[]?` | Tool-name whitelist from the scheduled task or chat session. Pass through to your CLI if it accepts one; otherwise advisory. |
| `signal`       | `AbortSignal`        | Aborts on cancel + on host shutdown. Wire to `treeKill(child.pid)`.                                                          |
| `logger`       | `PluginLogger`       | Scoped to this run. Output lands in `runner-<id>-<runId>.log`.                                                               |
| `cliPath`      | `string?`            | The user's override from Settings → Runners. Fall back to your default (typically `id`) when unset.                          |

### `ChatEvent`

```ts
type ChatEvent =
  | { type: 'text_delta'; delta: string }
  | { type: 'tool_call'; id: string; name: string; input: unknown }
  | { type: 'tool_result'; id: string; output: unknown; isError?: boolean }
  | { type: 'usage'; inputTokens: number; outputTokens: number; cachedInputTokens?: number }
  | { type: 'done'; stopReason: StopReason; error?: string }
  | { type: 'error'; message: string };

type StopReason =
  | 'stop'
  | 'tool_use'
  | 'max_tokens'
  | 'cancelled'
  | 'runner_error'
  | 'runner_not_installed';
```

Identical to the `ChatEvent` provider adapters emit — runners and providers
share the canonical streaming format. See the [event translation table](./authoring/#event-translation)
in the authoring guide for the CLI-signal → event mapping.

### `RunnerInstallStatus`

```ts
interface RunnerInstallStatus {
  installed: boolean;
  version?: string;
  detail?: string;
}
```

`detail` is shown verbatim in the Settings status pill tooltip when
`installed` is false. Use it to point the user at the fix (install
command, path override location).

### `RunnerRegistry`

Host-side registry. Plugin authors don't construct one — the host owns the
singleton, and `host.registerRunner(...)` is the supported path. Listed
here for completeness and for in-tree contributors:

```ts
class RunnerRegistry {
  register(runner: SubagentRunner): void;
  unregister(id: string): boolean;
  get(id: string): SubagentRunner | undefined;
  list(): readonly SubagentRunner[];
  has(id: string): boolean;
}
```

| Method             | Description                                                                     |
| ------------------ | ------------------------------------------------------------------------------- |
| `register(runner)` | Adds the runner. Throws `RunnerAlreadyRegisteredError` if `runner.id` collides. |
| `unregister(id)`   | Removes a runner by id. Returns `true` if something was removed.                |
| `get(id)`          | Look up a single runner.                                                        |
| `list()`           | All currently registered runners, in insertion order.                           |
| `has(id)`          | Cheap existence check used by the scheduled-task validator.                     |

---

### `RunnerAlreadyRegisteredError`

```ts
class RunnerAlreadyRegisteredError extends Error {
  readonly runnerId: string;
}
```

Thrown by `RunnerRegistry.register()` and bubbled up by
`host.registerRunner()` when two runners try to claim the same `id`.
`runnerId` is the conflicting id.
