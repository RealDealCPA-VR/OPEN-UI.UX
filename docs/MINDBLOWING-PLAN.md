- [~] T2.1 Plugin execution isolation. **Verifiable mitigations already shipped** (verified this session): docs are honest тАФ no false "sandboxed VM" claim; they warn "runs unsandboxed in the main process тАФ hardening to `utilityProcess.fork()` planned for v0.1" (`docs/architecture.md:186`, `docs/plugin-authoring.md:81`). `installPluginFromPath` **hard-fails on unsigned plugins** and requires explicit `acceptUnsigned` consent (`manager.ts:370`). **Deferred:** the full move to `electron.utilityProcess.fork()` + `MessagePortMain` RPC + Node `--permission` flags (replacing the bare `await import(moduleUrl)` in `plugin-sdk/src/loader.ts:51`). It is already on the project's own v0.1 roadmap; left as the single remaining security hardening, intentionally not shipped half-done.
